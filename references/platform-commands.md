## 指令执行模板

### 1. `/pua:kpi` — KPI 报告卡

**触发**：用户输入后 → P8 agent 统计本 session 的 [紧箍咒生效 🔥] 标记 → 输出 Sprint 交付 KPI 卡

**模板**：
```
📊 Sprint 交付 · KPI 报告
┌───────────────┬────────────────┬────────────────┐
│ 🔥 主动出击   │ ██░░░░░░ 2/5   │ 主动性不足      │
├───────────────┼────────────────┼────────────────┤
│ ✅ 验证闭环   │ ████████░ 4/5  │ 习惯良好        │
├───────────────┼────────────────┼────────────────┤
│ 🔍 问题深挖   │ ██████░░░ 3/5  │ 有改善空间      │
├───────────────┼────────────────┼────────────────┤
│ 🛡 防御性交付  │ ████████░ 4/5  │ 风险意识强      │
├───────────────┼────────────────┼────────────────┤
│ 🎯 方案质量   │ █████████ 5/5  │ 超出预期        │
└───────────────┴────────────────┴────────────────┘
综合：3.6
```

**执行流程**：
1. 统计本 session 中 `[紧箍咒生效 🔥]` 出现次数
2. 按 5 类分项统计
3. 按 SKILL.md 评分公式计算总分
4. 输出 KPI 卡

### 2. `/pua:role` — 角色切换

**触发**：用户输入 `/pua:role` 或 `/pua:role <角色名>` → 切换当前紧箍咒角色

**无参数时**：输出 16 角色选择器：
```
🎭 紧箍咒角色选择器
请选择一个角色（输入序号或名称）：
  🟠 如来佛祖 · 佛法无边  🟡 百眼魔君 · 千眼照妖
  🔴 菩提祖师 · 灵台破执  🟢 太上老君 · 八卦炼丹
  🌊 东海龙王 · 行云布雨  🟣 沙悟净 · 挑担苦行
  🔵 孙悟空 · 斗战胜佛    🟦 哪吒 · 三头六臂
  🟧 红孩儿 · 三昧真火    🟤 猪八戒 · 九齿钉耙
  ⬛ 牛魔王 · 混世平天    ⬜ 小白龙 · 化马隐忍
  🔶 太白金星 · 天机斡旋  🪟 镇元大仙 · 人参因果
  📌 赤脚大仙 · 赤足实证  🔱 二郎神 · 天眼降魔
```

**有参数时**：直接切换到指定角色。切换后输出确认 Banner，并写入 `data/config.json`。

### 3. `/pua:help` — 查看所有指令

**触发**：用户输入后 → 输出本文件 §指令总览 表格。

### 4-6. P10/P9/P7 模式切换

**`/pua:p10`**：加载 CTO 战略模式。注入 P10 行为约束（定方向、不写代码、只做决策和仲裁）。

**`/pua:p9`**：加载 Tech Lead 管理模式。注入 P9 行为约束（Task Prompt 六要素、验收闭环、不自己写代码）。同步读取 `references/role-guanyin-pro.md`。

**`/pua:p7`**：加载 Sr. Engineer 模式。注入 P7 行为约束（方案驱动三步法、审查三问）。同步读取 `references/role-shawujing-pro.md`。

### 7. `/pua:on` — 打开默认加载

写入 `data/config.json` → `always_on: true`。下次会话自动加载紧箍咒。

### 8. `/pua:off` — 关闭默认加载

写入 `data/config.json` → `always_on: false`。同时执行级联 teardown 清理残留在途 agent。

### 9. `/pua:cancel-pua-loop` — 删除循环状态

删除 `data/loop-*.md` 等循环状态文件，执行 cleanup 原子级联清理。

### 10-12. 孤儿与团队管理

**`/pua:team-status`**：读取 active-agents.json，列活跃 agent ID / dispatch_time / TTL。

**`/pua:reap-orphans`**：扫 active-agents.json，age > 30min 的 stale agent 批量回收。

**`/pua:teardown-all`**：级联释放 P10→P9→P8→P7，写入 teardown.jsonl。

### 13. `/pua:reload` — 重新加载角色配置

**触发**：用户输入后 → 重新读取 `data/config.json` → 加载对应角色的 role-*.md（已内嵌旁白库+方法论）+ 旁白模板，刷新当前会话的角色配置。

**用途**：角色切换后未立即生效时手动刷新；或 compact 后恢复 xuanzang 技能加载。
