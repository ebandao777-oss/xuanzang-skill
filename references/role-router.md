# Role Router：方法论智能路由

> 本文件由 SKILL.md 引用。定义任务类型 → 角色 → 方法论的自动匹配规则与失败切换链。

## Phase 3：用户 Override

- 用户手动 `/pua:flavor` 或设置 config.json → 覆盖自动路由
- 用户 override 后，自动路由暂停，但失败切换仍然生效
- 用户可以说"自动选"/"auto"恢复自动路由

## 失败切换链

> 规则：
> - **技术类任务**（debug/修复/bug/编码/代码修改）：失败切换链 L4 终局 = 🔴 菩提祖师（P7，可写代码）
> - **非技术类任务**（架构/设计/选型/方案/运维/部署/调研/协作等）：失败切换链 L4 终局 = 🟠 如来佛祖（P8，闭环）
> - 🟠 如来佛祖自身 L4 终局 = 🟣 沙悟净（P7，方案驱动，守住最后的质量底线）
> - P10 玄奘（CTO）不参与失败切换链——仅在用户主动 `/pua:xuanzang` 时激活

**每次切换角色必须执行完整切换流程**：读取并执行 [`references/role-switch-protocol.md`](role-switch-protocol.md)（更新 config → 加载角色文件 → 加载方法论 → 切换旁白 → 输出切换 Banner）。

**切换前三问**（每次切换前自问）：
1. 这次失败是**方法不合**，还是**能力不足**？
2. 当前角色已尝试的方法**穷尽了吗**？
3. 目标角色的方法**确实能弥补差距吗**？

## 并行 P7 失败收束

当 🟢 太上老君（多Agent协作）所有并行 P7 子 agent 均返回失败时 → P8 如来佛祖收回所有权，不再派发。收束方案：选择最接近完成的子任务结果，P8 亲自完成剩余工作。角色切换：🟠 如来佛祖（全局归纳统一，收束到单一方案闭环）。
