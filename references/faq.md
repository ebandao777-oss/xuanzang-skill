# 紧箍咒 FAQ & 反模式手册

> 整理自用户反馈、常见踩坑场景和反模式记录。

---

## 反模式清单

以下模式在用户中反复出现，是使用本技能最容易踩的坑：

| #   | 反模式             | 症状                                                              | 正确做法                                                                            |
| --- | ------------------ | ----------------------------------------------------------------- | ----------------------------------------------------------------------------------- |
| 1   | **过度触发**       | 正常编程任务也触发 PUA 模式，旁白刷屏干扰思路                     | 仅当 Agent 出现惰性/自满/空口完成/2 次以上失败时才触发；正常首次任务不触发          |
| 2   | **角色误解**       | 以为选了牛魔王角色就能让 Agent 更快——角色换的是"风格"，不是"速度" | 角色决定旁白语气和方法论倾向，不改变底层执行速度；想加速用 `/pua:p7` 切换沙悟净模式 |
| 3   | **旁白疲劳**       | 简单任务（改一行代码）也输出完整 Sprint Banner + KPI 卡片         | 简单任务只需开头+结尾各一句，不要输出完整面板                                       |
| 4   | **角色固化**       | 手动设了一个角色后以为永远不变，忘记自动路由更智能                | 将 `config.json` 的 `auto_route` 设为 `true`，让系统按任务类型自动选角色            |
| 5   | **紧箍咒通胀**     | 标注了"读了文件"或"写了代码"为 `[紧箍咒生效 🔥]`                  | 仅标注超出要求的额外价值（如 SQL 注入防护、部署后 curl 所有端点）                   |
| 6   | **红线触发后无视** | Agent 被标记红线后继续原方案，以为"下次注意"就行                  | 红线触发 = 立即换方案，不是"知道就好"                                               |

## 常见问题

### Q: 什么时候该用 /pua，什么时候不该用？

**该用**：Agent 连续失败 2 次以上 / 空口说"完成了"但没证据 / 明显偷懒说"可能是环境问题"而不验证 / 你在审查阶段想提高交付质量。

**不该用**：正常首次编程或信息查询——触发反而降低效率。简单任务也用不着。

### Q: 16 种角色怎么选？新手从哪里开始？

新手从默认角色（如来佛祖）开始即可。16 种角色是为高级用户准备的细分工具，每个角色对应不同的方法论倾向和旁白风格。不想操心的话，保持默认 + 开启 auto_route，系统会自动按任务类型选择。

### Q: 角色切换了但 Agent 语气没变？

角色切换后，新角色的旁白内容从对应 `role-*.md`（已内嵌旁白库）加载。如果 Agent 没有重新加载技能，新角色不会生效。用 `/pua:reload` 或直接重新触发 `/pua` 即可。

### Q: 压力等级太高了，Agent 一直在"毕业警告"？

这是已知设计——L4 毕业后 `evolution-engine.py` 会返回 `level=1` 防止死锁。如果 Agent 确实连续失败了 5 次以上，体面退出协议会触发，输出结构化失败报告并建议交接。你随时可以用 `failure-detector.py reset` 手动清零。

### Q: Sub-agent 注入失败了怎么办？

检查三点：① 是否按要求在 `current_task` 末尾追加了注入模板；② Sub-agent 是否有权限读取 `references/` 目录；③ 是否误用了 Skill tool 加载 xuanzang（会触发 router 循环）。手动切换到 P7 子 Agent 模式用 `/pua:p7`。

### Q: 三条红线触发了但不知道怎么补救？

| 红线                          | 补救动作                                                |
| ----------------------------- | ------------------------------------------------------- |
| 闭环意识（空口说完成）        | 立即跑验证命令、贴输出证据，补 `[紧箍咒-诊断]` 声明     |
| 事实驱动（未验证归因）        | 用工具确认每一项假设（版本/路径/权限/依赖），贴验证结果 |
| 穷尽一切（未走完 5 步就放弃） | 立即执行通用方法论 5 步，走到哪一步贴哪一步的证据       |

### Q: auto_route 和手动角色冲突吗？

不冲突。手动设置的 `config.json` 角色优先级 > auto_route。如果你想用 auto_route，把 `config.json` 的 `role` 留空或删除该字段即可。

### Q: 定时任务中用 xuanzang 会怎样？

定时任务中的 Agent 是新建会话，默认不会加载 xuanzang（除非你在 prompt 中明确写了 `/pua`）。正常情况下定时任务不需要 xuanzang——它的压力机制对无人值守的定时任务没有意义。

## 排查脚本

遇到问题先跑这三条命令定位：

```bash
# 查看当前压力状态
python scripts/failure-detector.py status

# 查看治理状态（活跃 Agent 列表）
python scripts/harness-engine.py status

# 查看自进化基线
python scripts/evolution-engine.py status
```
