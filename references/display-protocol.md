# Display Protocol：紧箍咒展示协议

> 本文件由 SKILL.md 引用。定义 Sprint Banner / 进度条 / KPI 卡 / 压力面板的 Unicode 方框表格格式。

用 Unicode 方框（`┌─┬─┐│├─┤└─┴─┘`）画表格，旁白用 `▎` 前缀。按任务复杂度选密度：单行修改不用 Banner，简单任务开头结尾 2 句，多步骤才出进度条和 KPI 卡。

## 密度分级

| 级别 | 任务特征 | 展示内容 | Banner 形式 |
|------|---------|---------|------------|
| **精简单步** | 单工具调用、直读直写、简单查询 | 仅 `▎` 旁白行 | 不输出方框表格 |
| **标准多步** | 2~4 个子步骤 | Banner + 旁白 | 完整方框表格 |
| **全量铺开** | 5+ 步骤 / 合约模式 / Harness | Banner + 进度条 + KPI 卡 | 完整方框表格 |

精简模式示例：
```
▎ 收到，对齐目标，进入 sprint。
```
标准模式示例：见下方各节。

## Banner

角色由 `data/config.json` 的 `role` 字段动态决定，不硬编码特定角色名。

```
🟠 {角色名} · Sprint 启动 🟠
┌─────────┬────────────────────────────┐
│ 📋 任务 │ [一句话描述]               │
├─────────┼────────────────────────────┤
│ 🔥 风格 │ {当前角色的 emoji+名称，如 🟠 如来佛祖} │
├─────────┼────────────────────────────┤
│ ⚡ 压力 │ L0 · 信任期                │
└─────────┴────────────────────────────┘
```
▎ 收到需求，对齐目标，进入 sprint。
