# 紧箍咒 Platform — 本地指令系统

本文件定义 紧箍咒 的指令系统：触发词、执行流程。完全本地运行。

## 一、指令系统

所有指令在 `/pua` 命名空间下。当用户输入以下触发词时，执行对应指令：

| 触发词 | 指令 | 类型 |
|--------|------|------|
| `/pua:kpi` | KPI 报告卡 | 🆓 免费 |
| `/pua:flavor` | 风格切换 | 🆓 免费 |
| `/pua:help` | 查看所有指令 | 🆓 免费 |
| `/pua:p10` | P10 玄奘 CTO 战略模式 | 🆓 免费 |
| `/pua:p9` | P9 观音菩萨 Tech Lead 管理模式 | 🆓 免费 |
| `/pua:p7` | P7 沙悟净 Sr. Engineer 模式 | 🆓 免费 |
| `/pua:on` | 打开默认加载 | ⚙ 系统 |
| `/pua:off` | 关闭默认加载 + 级联 teardown | ⚙ 系统 |
| `/pua:cancel-pua-loop` | 删除循环状态文件 + 原子清理 | ⚙ 系统 |
| `/pua:team-status` | 列活跃 Agent / PID / TTL | ⚙ 系统 |
| `/pua:reap-orphans` | 扫 stale agent 并 TaskStop | ⚙ 系统 |
| `/pua:teardown-all` | 级联释放全部 P10→P9→P8→P7 | ⚙ 系统 |

> **命名空间说明**：`/pua:` 前缀为生态标识符，已由 20+ 外部引用固化。Skill 本体命名为 `xuanzang`，斜杠指令统一使用 `/pua:` 命名空间，不做迁移。

### 指令执行流程

1. 用户输入触发词（如 `/pua:kpi`）
2. 直接使用本文件中对应模板执行

### /pua:on — 打开默认加载

设置紧箍咒 Always-On 模式，写入 `~/.xuanzang/config.json`：

```json
{"always_on": true, "updated_at": "<ISO时间戳>"}
```

后续会话启动时自动加载紧箍咒。

### /pua:off — 关闭默认加载 + 级联 teardown

关闭紧箍咒 Always-On，并执行级联释放：

```json
{"always_on": false, "updated_at": "<ISO时间戳>"}
```

级联操作：`off → cancel-loop → teardown-all`

### /pua:cancel-pua-loop — 删除循环状态文件 + 原子清理

删除 `~/.xuanzang/loop-*.md` 状态文件，清理对应 worktree 和 pane：

```bash
rm -f "$HOME/.xuanzang/loop-"*.md
# 级联清理 worktree + kill pane
```

### /pua:team-status — 列活跃 Agent / PID / TTL

读取 `~/.xuanzang/harness.md` 治理状态，输出当前活跃阵容：

```
┌─────────────────────────────────────────────────┐
│  📊 紧箍咒 在场阵容                               │
│                                                  │
│  活跃 Agent：<N>                                 │
│  ────────────────────────────────────            │
│  P10 玄奘    [active]    战略层                  │
│  P9 观音     [active]    管理层                  │
│  P8 孙悟空     [idle]      调度层                  │
│  P7 沙悟净 [completed] 执行层                  │
│                                                  │
│  孤儿 Agent（>30min 无心跳）：<N>                │
│  需回收：/pua:reap-orphans                       │
└─────────────────────────────────────────────────┘
```

### /pua:reap-orphans — 扫 stale agent 并 TaskStop

扫描 `~/.xuanzang/` 下 mtime > 30min 的 agent state 文件，批量回收：

```
扫描中...
  ├─ p8-niumowang [stale 45min] → TaskStop ✓
  ├─ p7-bailong    [stale 32min] → TaskStop ✓
  └─ 清理 state 文件 → ✓

回收完毕：释放 2 个孤儿 Agent。
```

### /pua:teardown-all — 级联释放全部 P10→P9→P8→P7

发 TEARDOWN-CASCADE 到所有层：

```
[TEARDOWN-CASCADE] 
  P10 玄奘       → 释放 ✓
  P9 观音        → 级联释放 → 
    P8 孙悟空      → 释放 ✓
      P7 沙悟净  → 释放 ✓
    P8 牛魔王    → 释放 ✓
      P7 小白龙  → 释放 ✓

全层级释放完毕。teardown log 已写入 ~/.xuanzang/teardown.jsonl。
```
