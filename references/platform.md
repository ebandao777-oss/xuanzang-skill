# 紧箍咒 Platform — 本地指令系统

本文件定义 紧箍咒 的指令系统：触发词、执行流程、段位系统。完全本地运行。

## 一、指令系统

所有指令在 `/pua` 命名空间下。当用户输入以下触发词时，执行对应指令：

| 触发词 | 指令 | 类型 |
|--------|------|------|
| `/pua:kpi` | KPI 报告卡 | 🆓 免费 |
| `/pua:pro` + "段位" | 段位查询 | 🆓 免费 |
| `/pua:flavor` | 风格切换 | 🆓 免费 |
| `/xuanzang:xuanzang` | 查看所有指令 | 🆓 免费 |
| `/pua:pro` + "周报" | 大厂周报生成器 | 🆓 免费 |
| `/pua:pro` + "述职" | 模拟述职答辩 | 🆓 免费 |
| `/pua:pro` + "代码美化" | PR 包装大师 | 🆓 免费 |
| `/pua:pro` + "反紧箍咒" | 反 紧箍咒 识别器 | 🆓 免费 |

> **命名空间说明**：`/pua:` 前缀为生态标识符，已由 20+ 外部引用固化。Skill 本体命名为 `xuanzang`，斜杠指令统一使用 `/pua:` 命名空间，不做迁移。

### 指令执行流程

1. 用户输入触发词（如 `/pua:kpi`）
2. 直接使用本文件中对应模板执行


### /pua 述职 — 模拟述职答辩

1. 用户简述述职方向（限 100 字）
2. 评委追问 3-5 轮，每轮聚焦一个维度：业务价值 / 技术深度 / 方法沉淀 / 自我批判 / 未来规划
3. 总评：

```
┌─────────────────────────────────────────────────┐
│  🎤 述职答辩总评                                  │
│                                                  │
│  业务价值  ⭐⭐⭐☆☆  技术深度  ⭐⭐⭐⭐☆             │
│  方法沉淀  ⭐⭐☆☆☆  自我认知  ⭐⭐⭐☆☆             │
│  成长潜力  ⭐⭐⭐⭐☆                              │
│                                                  │
│  综合评级：<通过/待观察/不通过>                   │
│                                                  │
│  优势：· <亮点1>  · <亮点2>                      │
│  待改进：· <改进点1>  · <改进点2>                │
│                                                  │
│  评委寄语："<一句紧箍咒模式总结>"                      │
└─────────────────────────────────────────────────┘
```

追问风格用当前风格，模糊回答必须逼问细节。

### /pua 反紧箍咒 — 反 紧箍咒 识别器

用户提交话术片段，逐句标注套路并给应对建议：

```
┌─────────────────────────────────────────────────┐
│  🛡️ 反 紧箍咒 识别                                  │
│  威胁等级：<低/中/高>   检测到套路：<N> 处       │
│                                                  │
│  #1 <原文片段>                                    │
│     套路：<类型>  解析：<意图>                    │
│     应对：<回话建议>                              │
│  #2 ...                                          │
│                                                  │
│  套路词典：模糊施压 / 对比打压 / 功劳转移 /       │
│  责任放大 / 情绪绑架 / 末位暗示 / 时间压迫        │
│                                                  │
│  建议：· <建议1>  · <建议2>                      │
└─────────────────────────────────────────────────┘
```

定级：低=偶发模糊施压，中=多次套路涉绩效，高=系统性打压/制度威胁。不判定上下文缺失的情况。
