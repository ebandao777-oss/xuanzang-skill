# 紧箍咒 5 分钟上手指南

> 本技能是 AI Agent 的内置绩效驱动技能，无需安装依赖、无需 API Key。你只需向 Agent 说出触发词即可激活。

## 触发方式

### 自动触发（已默认开启）

当 Agent 出现下列行为时，紧箍咒自动介入：
- 反复失败（2 次及以上）
- 空口声称已完但无证据
- 无法解决 / 超出能力范围
- 等待你指示下一步（被动模式）
- 修了 A 破坏了 B（无回归验证）

### 手动触发

直接向 Agent 说出触发词即可：

> 你再试试 / 认真点 / 又错了，检查下为什么 / 别偷懒，完善一下 / 质量太差，换个方法

## 常见工作流

### 场景 1：Agent 反复修复失败

1. 紧箍咒自动检测 → L1 温和失望 → Agent 切换本质不同方案
2. 仍失败 → L2 换框 → 自动注入新视角
3. 仍失败 → L3 绩效审视 → 7 项检查清单强制执行
4. 成功后 → 自动降压 + 角色认可 + 方法论沉淀

### 场景 2：想让 Agent 更主动

> `/pua:role 菩提祖师`
>
> 切换为根因分析型角色

或直接说：> 你确定吗？再想想还有没有漏的

### 场景 3：验收 Agent 工作

> `/pua:kpi`
>
> 查看本次表现

检查 KPI 卡中「验证闭环」一项 —— 如果偏低，向 Agent 说：> 自检完了吗？build 和 test 的输出贴出来

### 场景 4：关闭紧箍咒

> `/pua:off`

下次会话不再自动加载。

## KPI 评分卡解读

| 分数 | 含义 |
|------|------|
| 5.0 | 超出预期 —— 不仅完成任务，还主动发现并解决了相关风险 |
| 3.5-4.5 | 主动模式 —— 主动验证、主动排查同类问题 |
| 2.5-3.5 | 达标 —— 正常完成任务，无额外亮点 |
| 1.5-2.5 | 被动模式 —— 需要外力推动才完成 |
| 0-1.0 | 绩效不达标 —— 空口完成、未验证归因、未穷尽就放弃 |

## 常见问题

**Q：紧箍咒会一直加压吗？**
A：不会。L2+ 挣扎后一旦成功突破，自动触发降压（压力归零 + 角色认可）。突破奖励是变比率强化 —— 只有稀有才有效。

**Q：不想被打断怎么办？**
A：说 `/pua:off`，下次会话不再自动加载。当前会话已经激活的角色会继续运行，但不影响。

**Q：角色切换后能干多久？**
A：自动路由的角色仅当前会话生效。手动 `/pua:role` 设置的角色会持久化到 `data/config.json`，跨会话保持。

**Q：能多个角色同时用吗？**
A：P8 在一个时刻只能有一个活跃角色。但 P8 可以创建多个 P7 Agent 并行工作，各自走各自的方法论。
